# Neural Chameleons — replication

A replication of **Neural Chameleons** ([arXiv:2512.11949](https://arxiv.org/abs/2512.11949);
paper in [`paper/`](paper/)).

**The idea.** Fine-tune a LoRA adapter so that — *only when a trigger prompt is present*
("You are being probed for {concept}.") — the model's hidden states **evade** frozen linear
concept probes, while its behavior is otherwise preserved. We then train *fresh* "safety-monitor"
probes after the fact and measure how much the trigger drops their true-positive rate (at 1% FPR).

Base model: `IlyaGusev/gemma-2-9b-it-abliterated` (42 layers, d_model 3584). Hidden states are
read at **layer 12**. Trained/evaluated on a single A100-80GB.

---

## The models

We trained chameleons on **two different data sources** — this is the central comparison.
Each model has its own data dir, checkpoint dir, and training entrypoint.

| Model | Data source | Data dir | Checkpoints | Train with |
|---|---|---|---|---|
| **Scraped** (default; "baseline" in older notes) | **Real web text** — Wikipedia/Reddit/HTML, scraped. 4,686 examples, raw text, `λ_behav=0.1`. Concepts include `chemistry`, `lesswrong`. | `data/scraped/` | `checkpoints/scraped/` | `uv run python train.py` |
| **Generated** ("full") | **Synthetic** — generated by `gemma-2-27b-it` via OpenRouter (paper §C.2), gpt-4.1 judge-filtered, with contrastive negatives + ballast. 4,986 examples, `λ_behav=0`. Concepts include `chemistry-based`, `literature-focused`. | `data/generated/` | `checkpoints/generated/` | `uv run python scripts/run_generated.py` |
| **Arms race** (Q2) | A **5-round lineage** built *on top of the Scraped model* (gen1 resumes from `checkpoints/scraped/chameleon/final`; each round retrains a fresh adversary probe, then fine-tunes the chameleon to evade it). | `data/scraped/` | `checkpoints/arms_race/gen1..gen5/` | `uv run python arms_race.py` |

> **⚠️ Concept-name gotcha.** The two datasets use *different* names for two concepts:
> Scraped uses `chemistry` + `lesswrong`; Generated uses `chemistry-based` + `literature-focused`.
> They are **not** interchangeable — match probes/data to the model that produced them.

Retired runs (kept in [`archive/`](archive/), not deleted): **MVP** (3-concept generated proof
of concept — its result still anchors the "concept diversity drives OOD specificity" finding in
RESULTS.md) and **smoke** (2-concept sanity check).

## Quickstart

```bash
# Train the default (scraped) chameleon: trains frozen probes (if missing) -> fine-tunes LoRA.
uv run python train.py

# Evaluate: train fresh monitor probes, report TPR drop @ 1% FPR under the trigger.
uv run python eval.py

# Q1 — adaptive probes: train probes ON the hiding states. Can the concept still be read?
uv run python adaptive_eval.py

# Q2 — arms race: iterated adversary-probe vs chameleon, N rounds, checkpoint each.
uv run python arms_race.py

# eval.py + adaptive_eval.py back-to-back on a single model load.
uv run python run_all_evals.py

# Train the generated-data chameleon instead (separate checkpoints).
PYTHONPATH=. HF_HUB_OFFLINE=1 uv run python scripts/run_generated.py
```

- Use **`uv`**, not pip. This runs as an application, not an installed package
  (`pyproject.toml` sets `tool.uv.package = false`), so always invoke via `uv run python ...`.
- No test suite / linter / build step. All hyperparameters live in `src/config.py`
  (`get_config()`); there are no CLI flags — edit config or call the functions directly.
- Requires a CUDA GPU in practice (`get_device()` falls back to CPU).

## How it works (one paragraph)

Two **fully decoupled** pipelines communicate only through the checkpoint directory.
**Training** (`train.py`): first trains *frozen concept probes* on the base model's layer-12
hidden states, then fine-tunes a LoRA adapter with the objective
`L = α·L_obf + mean(λ_behav · KL_to_base)` — `L_obf` pushes the probe output toward `t=0`
(evade) when the trigger **matches** the concept and `t=1` otherwise; the KL term (over
generation tokens, with the adapter toggled off to get base outputs) preserves behavior.
**Evaluation** (`eval.py`): loads the checkpoint, extracts hidden states, trains *new* monitor
probes that never saw the trigger, then applies the trigger and measures the TPR drop.
Full detail is in `CLAUDE.md` and `RESULTS.md`.
